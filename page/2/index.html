计网1.1.1概念、组成、功能和分类

1.1.1 概念、组成、功能和分类

计算机网络的概念

  • 网络包含计算机网络。

  • 计算机网络：是一个将分散的、具有独立功能的计算机系统，通过通信设备与线路连接起来，由功能完善的软件实现资源共享和信息传递的系统

  • 计算机网络是互连的、自治的计算机集合。互连指的是通过有线或者无线的通信链路互联互通，自治指的是无主从关系。

计算机网络的功能

  1. 数据通信（连通性）
  2. 资源共享（软件、硬件、数据）
  3. 分布式处理：多台计算机各自承担同一工作任务的不同部分（Hadoop）
  4. 提高可靠性
  5. 负载均衡（分布式处理达到的效果）

计算机网络的组成

  1. 组成部分：硬件、软件、协议
    硬件：主机、链路、通信设备
    软件：安装在端系统上的应用软件
    协议（核心）：一系列规则和约定的集合
  2. 工作方式：边缘部分、核心部分
    边缘部分：用户可以直接使用的。用户可以进行通信。C/S方式和P2P方式。
    核心部分：为边缘部分进行服务的部分。由大量的网络和路由器组成。
  3. 功能组成：根据数据通信和资源共享这两大功能将计算机分为通信子网和资源子网
    通信子网：实现数据通信（物理层、数据链路层、网络层）
    资源子网：实现资源共享、数据处理（会话层、表示层、应用层）

计算机网络的分类

  1. 按照分布范围分：广域网WAN（交换技术）、城域网MAN、局域网WAN（广播技术）、个人网PAN
  2. 按使用者分：公用网、专用网
  3. 按交换技术分：电路交换、报文交换、分组交换
  4. 按拓扑结构分：总线型、星型（中间一个节点向外散播五条链路）、环形、网状型（常用于广域网）
  5. 按传输技术分：广播式网络（共享公共通信信道）、点对点网络（使用分组存储转发和路由选择机制）

修改Landscapes时遇到的一些问题

不知道去哪儿美化网站

一开始直接去github仓库中修改index.html和js脚本，后来发现每当push一个新的博客之后就会覆盖掉原来的样式，竹篮打水一场空。千万不要直接去github仓库中修改index.html代码！ hexo博客的生成是通过本地的.yml或js脚本push到github上自动生成的.html文件。如果要修改hexo博客，必须要在源头修改，也就是去本地的myblog文件夹中修改。

更改theme找不到文件夹

不要去myblog文件夹下的themes文件夹中去找！themes中存放的是下载的网络主题，而默认的lanscape主题并没有放在这里面。lanscape主题存放的路径为

node_modules/hexo-theme-landscape/..

在这里面可以通过修改.yml或者js脚本来美化网站。

Github卡顿问题

Github经常进不去或者崩溃怎么办？这里推荐大家使用UU加速器的学术资源来进行加速。亲测有效,并且免费！

莫名出现FATAL

如果没有修改本地文件，在hexo g -d的时候莫名出现FATAL问题，多半是网络波动。重启一下电脑试试吧！

localhost:4000和发布的网页不一样怎么办

我们对网页进行修改或者push一篇新的文章之后通过以下指令进行本地访问

hexo server

如果出现localhost:4000界面和在线网页不一样的情况，不如试试重新发布一遍，并强制清除浏览器缓存

hexo clean
hexo g -d

强制清除浏览器缓存

Ctrl+ 5

完美解决hexo插入图片无法显示问题

hexo博客中插入图片却无法显示，解决方案如下

1.打开Git Bash，安装图片路径转换插件
npm install https://github.com/CodeFalling/hexo-asset-image --save
2.更换/node_modules/hexo-asset-image/index.js为如下内容

'use strict';
var cheerio = require('cheerio');

// http://stackoverflow.com/questions/14480345/how-to-get-the-nth-occurrence-in-a-string
function getPosition(str, m, i) {
  return str.split(m, i).join(m).length;
}

var version = String(hexo.version).split('.');
hexo.extend.filter.register('after_post_render', function(data){
  var config = hexo.config;
  if(config.post_asset_folder){
        var link = data.permalink;
    if(version.length > 0 && Number(version[0]) == 3)
       var beginPos = getPosition(link, '/', 1) + 1;
    else
       var beginPos = getPosition(link, '/', 3) + 1;
    // In hexo 3.1.1, the permalink of "about" page is like ".../about/index.html".
    var endPos = link.lastIndexOf('/') + 1;
    link = link.substring(beginPos, endPos);

    var toprocess = ['excerpt', 'more', 'content'];
    for(var i = 0; i < toprocess.length; i++){
      var key = toprocess[i];
 
      var $ = cheerio.load(data[key], {
        ignoreWhitespace: false,
        xmlMode: false,
        lowerCaseTags: false,
        decodeEntities: false
      });

      $('img').each(function(){
        if ($(this).attr('src')){
            // For windows style path, we replace '\' to '/'.
            var src = $(this).attr('src').replace('\\', '/');
            if(!/http[s]*.*|\/\/.*/.test(src) &&
               !/^\s*\//.test(src)) {
              // For "about" page, the first part of "src" can't be removed.
              // In addition, to support multi-level local directory.
              var linkArray = link.split('/').filter(function(elem){
                return elem != '';
              });
              var srcArray = src.split('/').filter(function(elem){
                return elem != '' && elem != '.';
              });
              if(srcArray.length > 1)
                srcArray.shift();
              src = srcArray.join('/');
              $(this).attr('src', config.root + link + src);
              console.info&&console.info("update link as:-->"+config.root + link + src);
            }
        }else{
            console.info&&console.info("no src attr, skipped...");
            console.info&&console.info($(this));
        }
      });
      data[key] = $.html();
    }
  }
});

3.打开hexo文件夹下的_config.yml文件，修改下述内容
post_asset_folder: true
4.在.md文件中输入的文件路径

第一步安装插件后，new一个blog之后，_posts文件夹下不仅会自动生成名为blog.md的文件，还会生成一个名为blog的文件夹。文件夹内存放着blog.md内使用的图片资源。比如blog文件夹下存放着需要使用的图片picture.png，那么我们应该这么写调用图片的文件路径：
./blog/picture.png
Markdown就是：
![要替换的文字](./blog/picture.png)`
